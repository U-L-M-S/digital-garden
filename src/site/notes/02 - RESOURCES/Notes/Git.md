>Versionskontrollsystem.  
>Damit du mit Git arbeiten kannst, musst du **NUR** ein paar Befehle beherrschen (die Basics). 
>>Natürlich ist es immer gut, etwas fortgeschrittener zu sein, wenn du schneller und professioneller arbeiten möchtest.

Du kannst mit Git auf zwei Arten arbeiten: Offline(Local) oder Online(GitHub).

# Wie Git funktioniert (Workflow)
Git bewegt deine Dateien zwischen vier Bereichen: **Workspace → Stage → Local Repository → Remote Repository**. Jeder Befehl verschiebt Änderungen von einem Bereich in den nächsten.

>[!summary] Die Befehle im Überblick
>- `git add` → Workspace ins **Stage** (vormerken)
>- `git reset` → aus dem Stage zurück in den Workspace
>- `git commit` → Stage ins **Local Repository** (festschreiben)
>- `git push` → Local ins **Remote** (GitHub/GitLab)
>- `git fetch` → Änderungen vom Remote holen (noch nicht mergen)
>- `git pull` → `git fetch` + `git merge` in einem Schritt
